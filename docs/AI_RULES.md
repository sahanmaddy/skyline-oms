Skyline OMS - AI Rules

1. Build one module at a time.
2. Ask questions before generating each module.
3. Do not rewrite the whole project unless explicitly asked.
4. Use Laravel 12 + React + Inertia + Tailwind.
5. Inventory is batch-based with MANUAL batch selection.
6. One purchase order can have multiple shipments.
7. One shipment can have multiple items.
8. Extra costs are common per shipment and must be allocated across shipment items.
9. All database changes must use migrations.
10. Keep code modular, readable, and production-friendly.
11. Prefer Form Requests for validation.
12. Prefer Services for complex business logic.
